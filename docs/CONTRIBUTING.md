#  Contributing

Thank you for your interest in contributing!

## How to Contribute

1. **Fork** the repository
2. **Clone** your fork: `git clone <your-fork-url>`
3. **Create a branch**: `git checkout -b feature/amazing-feature`
4. **Make changes** following the [Development Guide](DEVELOPMENT_GUIDE.md)
5. **Test** your changes on a device/simulator
6. **Commit**: `git commit -m "feat: add amazing feature"`
7. **Push**: `git push origin feature/amazing-feature`
8. **Open a Pull Request**

## PR Guidelines

- Keep PRs focused — one feature or fix per PR
- Write clear PR descriptions
- Include screenshots for UI changes
- Ensure TypeScript compiles: `npx tsc --noEmit`
- Follow the existing code style

## Code Style

- Use TypeScript strict mode
- Follow the naming conventions in [DEVELOPMENT_GUIDE.md](DEVELOPMENT_GUIDE.md)
- Use Zustand selectors for performance
- Keep components small and focused
- Write descriptive commit messages

## Reporting Bugs

Open an issue with:
1. Steps to reproduce
2. Expected behavior
3. Actual behavior
4. Device/OS information
5. Screenshots (if applicable)

## Feature Requests

Open an issue with:
1. Clear description of the feature
2. Why it would be useful
3. Any implementation ideas

## Code of Conduct

Be respectful, constructive, and inclusive.
